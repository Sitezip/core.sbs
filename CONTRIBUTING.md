# Contributing to core.js

Thank you for your interest in contributing to core.js! We welcome contributions from the community.

## Getting Started

For detailed contributing guidelines, please visit our GitHub repository:

**[View Contributing Guidelines on GitHub](https://github.com/Sitezip/core.sbs/blob/main/CONTRIBUTING.md)**

## Quick Links

- **Issues**: [Report bugs or request features](https://github.com/Sitezip/core.sbs/issues)
- **Pull Requests**: [Submit your contributions](https://github.com/Sitezip/core.sbs/pulls)
- **Discussions**: [Join the conversation](https://github.com/Sitezip/core.sbs/discussions)

## Ways to Contribute

- 🐛 Report bugs
- 💡 Suggest new features
- 📝 Improve documentation
- 🔧 Submit pull requests
- 🎨 Create examples and tutorials
- 🧪 Write tests
- 💬 Help others in discussions

## Development Setup

```bash
# Clone the repository
git clone https://github.com/Sitezip/core.sbs.git
cd core.sbs

# Install CLI tools
npm install -g @core.sbs/create-core-app

# Start development server
core-dev
```

## Code Style

- Follow existing code patterns
- Keep functions small and focused
- Use meaningful variable names
- Add JSDoc comments for public APIs
- Test your changes thoroughly

## Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Questions?

If you have questions, please:
- Check the [documentation](https://core.sbs)
- Search [existing issues](https://github.com/Sitezip/core.sbs/issues)
- Start a [discussion](https://github.com/Sitezip/core.sbs/discussions)
